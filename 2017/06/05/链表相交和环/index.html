链表相交和环

链表相交(假设无环的情况下)

对于两个链表,如果他们相交的话，那么他们最后的一个节点一定是相同的，否则是不相交的。因此判断两个链表是否相交就很简单了，分别遍历到两个链表的尾部，然后判断他们是否相同，如果相同，则相交；否则不相交。

判断出两个链表相交后就是判断他们的交点了。假设第一个链表长度为len1，第二个问len2，然后找出长度较长的，让长度较长的链表指针向后移动|len1 - len2| (len1-len2的绝对值)，然后同时遍历两个链表，判断节点是否相同即可。

一个有环的链表,如何判断有环,如何得到环的入口点

链表中有环，其实也就是自相交。我们用两个指针pslow和pfast从头开始遍历链表，pslow每次前进一个节点，pfast每次前进两个结点，若存在环，则pslow和pfast肯定会在环中相遇，若不存在，则pslow和pfast能正常到达最后一个节点（实际上是到达NULL）。

当pfast若与pslow相遇时，pslow肯定没有走遍历完链表，而pfast已经在环内循环了n圈(1<=n)
假设pslow走了s步，则pfast走了2s步（pfast位置还等于s 只是在环上多转的n圈）
设环长为r，则：
2s = s + n r ======> s= n r
设整个链表长L，入口环与相遇点距离为x，起点到环入口点的距离为a
a + x = n r 则
a + x = (n – 1) r + r = (n - 1) r + L - a
a = (n - 1) r + (L – a – x)
(L – a – x)为相遇点到环入口点的距离,由此可知:
从链表头到环入口点等于: (n-1)次循环内环 + 相遇点到环入口点
于是我们从链表头、与相遇点分别设一个指针，每次各走一步，两个指针必定相遇，且相遇第一点为环入口点。

两链表,在可能有环的情况下,判断是否相交,如果相交,得到交点

对于两个链表,可能有以下几种情况:

  • 两个链表都没有环
    • 两链表不相交
    • 两链表相交
  • 其中一个链表有环
    • 两链表肯定不相交
  • 两个链表都有环
    • 两链表不相交
    • 两链表相交
      • 两链表相交点在环之前或交点和环入口点一样,,即,两链表环入口点一样
      • 两链表相交点在一个环内
  1. 判断L1是否有环,获取出长度len1,如果有环,获取环入口点e1,如果没有,获取尾节点f1
  2. 判断L2是否有环,获取出长度len2,如果有环,获取环入口点e2,如果没有,获取尾节点f2
  3. 如果两个链表都没有环
    1. 判断尾节点是否一样,不一样则判断不相交,结束
    2. 一样: 两链表重新遍历: 长链表先前进(MAX(len1,len2) - MIN(len1,len2)),之后两链表同时前进,直到找到同一个节点
  4. 如果一个链表有环,另一个没有, 则判断不相交,结束
  5. 如果两个都有环
    1. 如果e1 == e2, 两链表重新遍历: 长链表先前进(MAX(len1,len2) - MIN(len1,len2)),之后两链表同时前进,直到找到同一个节点
    2. 如果e1 != e2, 遍历其中一个环,判断另一个环的入口在不在这个环中
    3. 如果在不在同一个环中, 则判断不相交,结束
    4. 如果在同一个环中对于L1来说,e1就是交点,对于L2,e2就是交点
